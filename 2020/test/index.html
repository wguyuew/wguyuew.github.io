搭建准备工作

  • Hugo –> 博客框架

Git账号获取

自行百度或谷歌（狗头保命）

Git

  1. 官网下载安装

  2. 打开cmd(win+R)输入

    git version

  3. git 部署个人账号和邮箱 右键打开Git Bush here

git init git config username “你的gitbub用户名” git config useremail “邮箱” git config username //进行检查 git config useremail //进行检查

若出现版本号则成功

Hugo

下载安装

  1. 官网下载安装

  2. 将安装包解压到你想保存的路径（记住其中的.exe文件路径）

  3. 将上述路径加入到环境变量中去

    • 此电脑 选中属性

    • 打开高级系统设置

    • 打开环境变量

    • 找到path点入添加即可

  4. 打开cmd(win+R)输入

    hugo version

若出现版本号则成功

创建网站

  1. 选择你想保存的文件夹，进入目录输入

    hugo new site xxxx(文件名例：myblog)

    输入后可检查myblog中是否含有一下文件

    ├── archetypes # 内容类型，在创建新内容时自动生成内容的配置
    ├── content # 网站内容，Markdown 文件
    ├── data
    ├── layouts # 网站模版，选择主题后会将主题中的 layouts 文件夹中的内容复制 到此文件夹中
    ├── static # 包含 CSS、JavaScript、Fonts、media 等，决定网站的外观。选择主题后会将主题中的 ststic 文件夹中的内容复制到此文件夹中
    ├── themes # 存放主题文件
    └── config.toml # 网站的配置文件

  2. 下载主题

  • 通过git submodule add以Git子模块的方式下载
    进入myblog，使用git init初始化后再下载主题

    cd themes
    git init git submodule add 主题网址

  • 通过git clone下载
    进入myblog

    cd themes git 主题网址

  • 更改配置文件
    下载完后，找到主题目录/themes/even/exampleSite下的配置文件config.toml，将该文件复制到网站文件夹下替换原来的config.toml，根据需求更改配置文件即可 tips config.toml介绍

    baseURL = “https://mogeko.github.io” # 里面的 baseurl 信息，填你的博客地址
    title = “XX`s Blog” # 浏览器的标题
    languageCode = “zh-cn” # 语言 hasCJKLanguage = true # 开启可以让「字数统计」统计汉字 theme = “LeaveIt” # 主题 (需要自己下载)
    paginate = 11 # 每页的文章数
    enableEmoji = true # 支持 Emoji
    enableRobotsTXT = true # 支持 robots.txt
    googleAnalytics = "” # Google 统计 id preserveTaxonomyNames = true

写博客

创建文章前，可以将主题文件夹下archetypes网站根目录的archetypes文件夹下，这个是文章的模板文件。

注意: 有的主题打开网站后看不到网站中创建的文章，是由于主题默认的路径为posts而不是post，此时可将文件夹名称post改为posts，创建文章时命令改为hugo new posts/my-first-post.md，也可以修改配置文件config.toml中指向的文件夹路径

启动

hugo server 打开浏览器去http://localhost:1313 浏览

遇到一个问题，在更换主题的时候，有时候打开本地网页会报错，先执行Ctrl + C关闭本地Hugo服务器再执行hugo server -D重启服务就行了

不同主题下的文章一般不能共用，在启动Hugo Server 时可能会报错，需要删除content目录下其它主题的文章

上传时 draft: true 改为false

与GitHub Pages联动

  1. 在GitHub建立repository,点击new repository
  2. 填入信息用户名.github.io(用户名须与github用户名内容一致)
  3. 建立完毕
  4. 在myblog目录下输入

git init
git remote add origin https://github.com/[Github 用户名]/[Github 用户名].github.io.git (刚才创建repository的地址) git add -A
git commit -m “[介绍，随便写点什么，比如日期]”
git push -u origin master

以后发布只需要执行

git add -A git commit -m “[介绍，随便写点什么，比如日期]”
git push -u origin master
